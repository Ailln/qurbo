# Alpha 测试数据说明

本目录包含两个 MIQP 样例数据集：

- `miqp_sample_A.npz`：小规模样例，适合读数据、调试算法流程和做暴力枚举校验。
- `miqp_sample_B.npz`：较大规模样例，适合作为 baseline 或启发式算法的主要测试对象。

两个文件都是 NumPy `.npz` 格式，字段结构一致。

## 字段含义

问题形式为：

```text
maximize    x^T Q x + c^T x + h^T y
subject to  A x + G y <= b
            B x <= b_prime
            x in {0, 1}^n
            y >= 0
```

各字段含义如下：

| 字段 | 形状 | 含义 |
| --- | --- | --- |
| `n` | 标量 | 二元变量 `x` 的数量。 |
| `p` | 标量 | 连续非负变量 `y` 的数量。 |
| `m1` | 标量 | 混合约束 `A x + G y <= b` 的数量。 |
| `m2` | 标量 | 纯二元约束 `B x <= b_prime` 的数量。 |
| `Q` | `(n, n)` | 二元变量的二次目标项矩阵。 |
| `c` | `(n,)` | 二元变量的线性目标项。 |
| `h` | `(p,)` | 连续变量的线性目标项。 |
| `A` | `(m1, n)` | 混合约束里二元变量的系数。 |
| `G` | `(m1, p)` | 混合约束里连续变量的系数。 |
| `b` | `(m1,)` | 混合约束右端项。 |
| `B` | `(m2, n)` | 纯二元约束系数。 |
| `b_prime` | `(m2,)` | 纯二元约束右端项。 |
| `optimal_value` | 标量 | 样例内置的最优目标值。 |
| `x_opt` | `(n,)` | 样例内置的最优二元解。 |
| `y_opt` | `(p,)` | 样例内置的最优连续解。 |
| `is_optimal` | 标量 | 内置解是否标记为最优。 |

| 数据集 | `Q` | `c` | `h` | `A` | `G` | `b` | `B` | `b_prime` |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| `miqp_sample_A.npz` | `(15, 15)` | `(15,)` | `(5,)` | `(5, 15)` | `(5, 5)` | `(5,)` | `(1, 15)` | `(1,)` |
| `miqp_sample_B.npz` | `(80, 80)` | `(80,)` | `(20,)` | `(20, 80)` | `(20, 20)` | `(20,)` | `(4, 80)` | `(4,)` |

## 稀疏度和结构

- 两个数据集的 `Q` 都是对称稠密矩阵，非零密度为 `1.0`。这意味着二元变量之间基本两两存在二次相互作用，局部翻转一个变量可能影响许多目标项。
- `A` 和 `G` 也都是稠密矩阵，非零密度为 `1.0`。固定一个 `x` 后，连续变量 `y` 的 LP 子问题虽然规模不大，但约束耦合比较完整。
- `B` 是 0/1 系数的纯二元约束矩阵，A 中 `B` 的非零密度约为 `0.5333`，B 中约为 `0.5031`，大约一半二元变量会参与每条纯二元约束。

## 难度直观理解

`miqp_sample_A.npz` 的二元变量只有 15 个，完整枚举只需要检查 `32,768` 种 `x`。每个候选 `x` 固定后，再解一个 5 个连续变量、5 条混合约束的线性规划子问题。因此 A 更像是调试样例：可以用暴力枚举校验 baseline 输出，也适合验证目标函数、约束检查和数据读取是否正确。

`miqp_sample_B.npz` 的二元变量有 80 个，完整搜索空间约为 `1.21e24`，已经不能依靠暴力枚举。它还包含 20 个连续变量、20 条混合约束和 4 条纯二元约束；同时 `Q/A/G` 都是稠密的，所以二元变量之间、二元变量与连续子问题之间的耦合明显更强。B 更适合测试启发式算法、局部搜索、QAOA 子问题分解等方法在大搜索空间中的表现。

内置最优解中，A 的 `x_opt` 有 8 个变量取 1，`y_opt` 有 3 个非零值；B 的 `x_opt` 有 41 个变量取 1，`y_opt` 有 4 个非零值。这可以作为检查输出解规模和稀疏性的参考。
